# Learning Plan

## Goal

以 AI × Web3 School 的课程为主线，从 Week 3 开始每天约 2–3 小时推进课程活动、Hackathon 项目方向和 proof-of-work；Week 4 Hackathon sprint 阶段预计提高到每天约 3–4 小时。

## Daily Routine

### Week 3｜每天 2–3 小时

1. **20–30 分钟**：查看 WCB Learning 页面，确认今日课程 / 活动 / 任务 / 打卡入口。
2. **60 分钟左右**：实时参加课程活动或观看回放，重点记录与 Hackathon、支付、Agent 执行、安全边界有关的内容。
3. **40–60 分钟**：整理 daily note、3 条有效笔记或 1 条 next action，并准备 WCB proof 草稿。
4. **40–60 分钟**：推进一个最小项目 artifact，例如方向卡、demo script、policy schema、README、分享稿或 Week 4 sprint plan。

### Week 4｜每天 3–4 小时

1. **30 分钟**：确认当天 sprint 目标、卡点和需要 mentor / co-learning 反馈的问题。
2. **90–120 分钟**：集中实现或调试 Hackathon demo 的一个核心路径。
3. **45–60 分钟**：补充 README、运行说明、audit log、截图或录屏素材。
4. **30–45 分钟**：整理当天 proof、提交草稿和次日任务。

## Path Options

- **最小路径**：完成今日课程 + 写 daily note + 生成打卡草稿。
- **推荐路径**：完成今日课程 + 一个小实验 / 小 PR / 项目推进点。
- **挑战路径**：把今日知识直接转化为 Hackathon 项目功能、架构设计或 demo。

## Weekly Review

每周至少复盘一次：

- 本周完成了哪些 proof-of-work？
- 哪些内容能进入 Hackathon 项目？
- 哪些 Handbook feedback 值得整理提交？
- 下周最重要的 1–3 个推进点是什么？
