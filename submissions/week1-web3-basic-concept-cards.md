# Week 1｜Web3 基础概念卡片

> 任务：整理至少 8 个 Web3 基础概念  
> 日期：2026-06-04  
> 说明：用自己的话整理，帮助理解测试网交易、合约调用、钱包确认和链上验证。本文不包含私钥、助记词、真实地址或任何敏感凭证。

## 1. Account / 账户

- 一句话解释：账户是链上资产和权限的载体，可以是普通外部账户，也可以是智能合约账户。
- 具体例子：用户的 EOA 钱包是一个账户，CAW Agent Wallet 也是一个受规则控制的账户。
- 常见误区：账户地址公开不等于可以公开私钥；控制权和可见性是两回事。

## 2. Address / 地址

- 一句话解释：地址是账户或合约在链上的公开标识，通常用于接收资产或调用合约。
- 具体例子：USDC 合约地址、用户钱包地址、Agent Wallet 地址都属于链上地址。
- 常见误区：地址看起来相似不代表安全；交互前要确认链、合约地址和来源。

## 3. Wallet / 钱包

- 一句话解释：钱包是管理账户、签名交易和展示资产的工具。
- 具体例子：用户用浏览器钱包连接 DApp，再确认一笔测试网 USDC 转账。
- 常见误区：钱包不是资产本身，资产记录在链上；钱包主要管理密钥和签名。

## 4. Private Key / 私钥

- 一句话解释：私钥是控制 EOA 的核心秘密，拥有私钥就等于拥有对应账户控制权。
- 具体例子：用户用私钥签名交易，证明自己同意转账或授权。
- 常见误区：私钥不能交给 AI、网页、后端日志或公开仓库；任何要求粘贴私钥的流程都应拒绝。

## 5. Seed Phrase / 助记词

- 一句话解释：助记词通常可以恢复一个钱包中的多个账户，是比单个私钥更敏感的恢复凭证。
- 具体例子：用户换设备时用助记词恢复钱包。
- 常见误区：助记词不是普通密码，不能截图、发给客服、放进云笔记或提交到 GitHub。

## 6. Signature / 签名

- 一句话解释：签名是账户对某个消息或交易的授权证明。
- 具体例子：用户签名登录 DApp，或签名确认一笔链上交易。
- 常见误区：不是所有签名都无风险；有些签名可能授权资产或权限，需要看清内容。

## 7. Transaction / 交易

- 一句话解释：交易是对链上状态的修改请求，例如转账、授权、调用合约。
- 具体例子：把测试网 USDC 转入 Agent Wallet，或调用合约执行 supply。
- 常见误区：交易发出后通常不可随意撤销；即使在测试网，也要养成检查链、地址、金额和函数的习惯。

## 8. Gas

- 一句话解释：Gas 是执行链上交易需要支付的计算资源费用。
- 具体例子：调用智能合约比普通转账通常消耗更多 gas。
- 常见误区：交易失败也可能消耗 gas；gas 低不代表交易一定划算，gas 高也不代表项目一定可信。

## 9. Smart Contract / 智能合约

- 一句话解释：智能合约是部署在链上的程序，可以按照预设规则管理资产、权限和状态。
- 具体例子：DeFi 协议合约可以接收 token、记录存款份额、发放收益。
- 常见误区：合约部署在链上不代表安全；需要看代码、审计、权限、历史和交互对象。

## 10. Testnet / 测试网

- 一句话解释：测试网是用于开发和实验的区块链网络，资产通常没有真实经济价值。
- 具体例子：Base Sepolia、Arbitrum Sepolia 可用于测试钱包连接、转账和合约调用。
- 常见误区：测试网可以降低资金风险，但不能替代安全设计；测试网代码上主网前仍需认真检查。

## 11. Block Explorer / 区块浏览器

- 一句话解释：区块浏览器是查看交易、地址、合约和事件日志的公开工具。
- 具体例子：通过 tx hash 查看一笔测试网 USDC 转账是否成功。
- 常见误区：区块浏览器显示成功只说明链上执行成功，不代表业务逻辑一定符合用户意图。

## 12. EOA / Smart Account / Multisig

- 一句话解释：EOA 由私钥直接控制；Smart Account 可以带规则；Multisig 通常需要多人确认。
- 具体例子：个人钱包常是 EOA；Safe 是常见多签；Agent Wallet 可以用智能账户和 policy 限制自动化行为。
- 常见误区：自动化不应建立在无限 EOA 权限上；涉及 AI Agent 时，更适合使用带限额、白名单和审计的账户设计。

## 和我的 AI × Web3 项目连接

YieldAgent 里的 Web3 边界可以这样理解：

```text
用户 EOA：负责人工确认和初始资金准备
Agent Wallet：只持有用户主动转入的小额测试网资金
Pact / Policy：限制 Agent 能操作什么、最多花多少、多久有效
Smart Contract / DeFi Recipe：真正发生链上交互的地方
Block Explorer / Audit Log：验证执行结果和复盘风险
```

我的理解是：Web3 给 AI Agent 带来的不是“无限自动执行”，而是可验证账户、可审计交易、可编程权限和可公开复查的执行记录。
