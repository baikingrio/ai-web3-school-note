## 今日学习主题：从 Demo 页面走向产品控制台

今天同步最新代码后，我最大的感受是：YieldAgent 开始从“多个功能页面”变成“一个更像产品的控制台”。

之前的页面结构更像：

```text
/                 landing
/wallet           钱包准备
/create-strategy  创建策略
/pacts            Pact 管理
/history          历史记录
/settings         设置
```

今天的结构更接近真实产品：

```text
/                         产品落地页
/dashboard                 控制台入口 / onboarding
/dashboard/create-strategy  创建策略
/dashboard/pacts            Pact 管理
/dashboard/history          Audit Trail
/dashboard/settings         设置
```

旧路径仍然保留为薄重定向，但主线被收敛到 `/dashboard/*`。这件事看起来像路由调整，其实对 Hackathon demo 很重要：评委或用户打开项目时，能更清楚地看到“先连接钱包，再完成 Agent Wallet 设置，再进入控制台”的路径。

## 技术栈 / 架构复盘

今天继续沿用 YieldAgent 已确定的技术栈：

- 前端：Nuxt 4 / Vue 3 / TypeScript / Tailwind CSS / shadcn-vue/ui 风格组件。
- 钱包连接：wagmi / viem。
- 执行层：Cobo Agentic Wallet（CAW）/ Pact。
- Agent / 策略层：Hermes runtime / Z.AI API 方向的自然语言策略解析 + deterministic fallback。
- 数据库 / 日志：SQLite，用来持久化策略、Pact、审计日志和收益快照。
- 部署：Vercel 前端 + 远程 Hermes Agent 主机后端 runtime。

今天几个比较关键的模块变化：

1. **Dashboard layout 和侧栏收敛**  
   新增 `app/layouts/dashboard.vue` 和 `DashboardSidebar`。控制台页面统一进入 dashboard layout，Agent Wallet 未就绪时不展示侧栏，而是展示 onboarding，引导用户先完成钱包准备。

2. **钱包连接状态变成产品入口的一部分**  
   新增 `AppWalletStatus.vue`，Header 上可以直接连接 / 断开钱包，并展示地址和网络状态。`require-wallet.global.ts` 会在访问 `/dashboard` 时检查 EOA 连接状态，不满足条件就回到首页。

3. **Agent Wallet onboarding 更集中**  
   `DashboardOnboarding.vue` 把 Agent Wallet 创建、导入、注资、重置、准备状态总结放到 dashboard 里。用户不是单独去 `/wallet`，而是在控制台入口完成前置准备。

4. **状态命名从 demo 转向 app**  
   `app/stores/demo.ts` 重命名为 `app/stores/app.ts`，`shared/types/demo.ts` 重命名为 `shared/types/app.ts`，`server/utils/demo-store.ts` 也改成 `app-store.ts`。这说明项目从 demo 原型逐步往 app 产品形态收敛。

5. **自然语言策略解析更安全**  
   `hermes-strategy-parser.ts` 增强了 daily amount / duration 的解析，比如“每日 1 USDC，持续一周”会被归一化为 `maxSpend = 7`，并且买入 ETH / swap 这类动作会被推断为更高风险等级。同时，Hermes 返回的参数仍然要经过 deterministic validation。

6. **收益曲线开始接链上仓位**  
   新增 `yield-snapshot.ts`，可以读取当前 supplied amount，计算与上次仓位快照的 delta，并写入 7 日收益曲线。这让 dashboard 的收益图不只是静态 mock，而是开始连接真实仓位变化。

## Hackathon 推进

今天的推进让 YieldAgent 的 demo 更像一个可以顺畅演示的产品：

1. **用户路径更清楚**  
   首页负责讲清楚产品价值，dashboard 负责承载实际操作。用户需要先连接钱包，再完成 Agent Wallet 设置，然后才能进入策略与 Pact 管理。

2. **控制台体验更完整**  
   侧栏、overview、create strategy、pacts、history、settings 都进入统一 dashboard 结构，后续 demo script 可以围绕这条路径展开。

3. **自然语言策略输入更可信**  
   Agent 可以理解“每日投入多少、持续多久、是否买 ETH”等描述，但最终仍然经过本地确定性校验，不会直接信任 LLM 输出。

4. **收益图从静态展示走向状态同步**  
   通过 `yield-snapshot.ts`，dashboard 的收益曲线开始和链上仓位变化关联起来，这对“收益策略 Agent”这个定位很重要。

5. **项目表达从 demo 收敛到 app**  
   代码命名从 demo -> app，不只是重命名，也说明 YieldAgent 的叙事从“演示脚本”逐步转成“可用控制台”。
