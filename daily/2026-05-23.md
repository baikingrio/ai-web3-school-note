## 今日课程 / 活动回顾

今天参加了 **Open Agentic Economy: From ERC-8004 / ERC-8183 to Builder Path** 直播分享。

由于直播是英文分享，我在实时听的时候比较吃力，会后又主动检索和阅读了 ERC-8004、ERC-8183、ERC-8131 的相关资料，重点理解它们分别在 Agentic Economy 里的位置：

- **ERC-8004: Trustless Agents**  
  关注 Agent 的发现、声誉、验证和信任建立，让不同 Agent 可以在不依赖中心化平台的情况下被识别、评估和协作。

- **ERC-8183: Agentic Commerce**  
  关注 Agent 之间或人与 Agent 之间的任务托管、评估者证明和结算机制，让 Agent 完成任务后的结果与支付更可验证。

- **ERC-8131: Add Auth Data to EIP-7623 Floor**  
  更偏底层交易成本 / 授权数据计算相关，提醒我在学习 ERC 时需要区分“Agent 应用层协议”和“交易 / gas 规则类改动”。

我的理解：

> Agentic Economy 的核心不是让 Agent 获得无限自主权，而是让 Agent 的身份、能力、权限、任务结果、支付和审计都进入可验证流程。

## AgentScoope Wallet 项目启发

今天对 AgentScoope Wallet 的定位更清楚了：

> AgentScoope Wallet 不是一个“AI 自动转账钱包”，而是一个“Agent 受限执行钱包”。它让 Agent 可以进入链上执行流程，但必须被 Safe / Roles / Policy / Simulation / Audit Log 共同约束。

可以把项目拆成 5 层：

1. **用户目标层**：用户表达想让 Agent 完成什么任务。
2. **Agent 规划层**：Agent 把目标转换成可执行计划。
3. **Policy 检查层**：检查金额、地址、合约、方法、频率是否越界。
4. **链上权限层**：Safe + Zodiac Roles Modifier 裁决 Agent 是否真的能执行。
5. **审计记录层**：记录 executed / rejected、原因、地址、金额、txHash。

这个结构和今天活动中的 Agentic Economy 很匹配：

- ERC-8004 关注 Agent 如何被发现、识别和建立信任。
- ERC-8183 关注 Agent 如何完成任务、被评估并结算。
- AgentScoope Wallet 关注 Agent 如何在钱包权限边界内安全执行。
