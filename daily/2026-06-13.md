## 今日学习主题：Demo Day 前的稳定性检查

昨天 YieldAgent 已经补上了很关键的一层：Vercel 部署下的 Postgres / Supabase 持久化与 deployment check。

今天我更关注的是：这些工程改动如何转成 Demo Day 前的检查清单。

我把 YieldAgent 的 Demo 风险拆成四类：

1. **状态是否稳定**
   - 钱包准备状态不能因为 serverless `/tmp` 丢失而消失。
   - 策略、Pact、执行日志应该能持久化。

2. **授权边界是否清楚**
   - Agent 主 Key 不能被当作 active Pact 执行凭证。
   - active Pact 执行需要 pact-scoped key。
   - 缺凭证时应该明确失败，而不是偷偷 fallback。

3. **前端是否能解释问题**
   - Dashboard 要能区分 ready、pending、failed、denied。
   - History / Recent Logs 要能看到真实执行或失败原因。
   - Settings deployment check 要帮助定位 CAW、TSS、Hermes、Postgres 哪一层有问题。

4. **公开材料是否统一**
   - 项目名统一使用 YieldAgent。
   - README、daily、学习笔记里不再混用旧项目名。
   - 公开仓库不记录会议链接、密码、API Key、私钥或真实凭证。
