### 5–15 分钟：理解 AI Agent 基础

重点理解以下概念：

- AI Agent
- Tool Calling / Function Calling
- 外部工具
- 任务规划
- 执行反馈

核心公式：

```text
AI Agent = 大模型 + 工具 + 任务规划 + 执行反馈
```

普通 AI 助手流程：

```text
用户提问 → AI 直接回答
```

AI Agent 流程：

```text
用户提问 → AI 判断任务 → 选择工具 → 调用工具 → 获取结果 → 分析结果 → 回复用户
```

思考例子：

如果用户说：

> 帮我看看这个钱包地址有没有风险。

普通 AI 只能解释分析方法；AI Agent 应该能够查询钱包余额、交易记录、授权状态、黑名单风险，并输出风险等级和建议。

### 15–30 分钟：学习 Tool Calling / Function Calling

Tool Calling 的理解：

> Tool Calling 是让大模型在需要时调用外部函数或 API，而不是凭空生成结果。

例如钱包查询工具：

```ts
getNativeBalance(address, chain)
getTokenBalances(address, chain)
getRecentTransactions(address, chain)
```

当用户问：

> 查询这个地址的 ETH 余额：0x123...

Agent 不应该直接猜测余额，而应该调用：

```ts
getNativeBalance("0x123...", "sepolia")
```

再根据工具返回结果生成自然语言回答。

需要理解的流程：

```text
用户输入
  ↓
大模型判断用户意图
  ↓
选择合适工具
  ↓
传入参数
  ↓
工具返回结果
  ↓
大模型整理成自然语言
```

今日重点判断：

- “什么是钱包授权？” → 可以直接解释。
- “帮我查这个地址余额” → 必须调用链上查询工具。
- “这笔交易安全吗？” → 应该调用交易模拟和风险分析工具。
- “帮我执行这笔转账” → 必须经过权限检查、风险判断和用户确认。

### 30–50 分钟：设计 Web3 钱包 Agent 工具

围绕 Hackathon 项目 **AgentScoope Wallet**，设计 4 类工具。

#### A. 钱包查询工具

```ts
getNativeBalance(address, chain)
getTokenBalances(address, chain)
getRecentTransactions(address, chain)
getNFTAssets(address, chain)
```

作用：

- 查询 ETH / 原生币余额
- 查询 ERC20 代币资产
- 查询近期交易记录
- 查询 NFT 资产

#### B. 风险分析工具

```ts
checkAddressRisk(address)
checkContractRisk(contractAddress)
checkTokenApprovalRisk(address)
analyzeTransactionRisk(txData)
```

作用：

- 判断地址是否为黑名单地址
- 判断合约是否可疑
- 检查用户是否存在高风险授权
- 分析交易是否可能导致资产损失

#### C. 交易模拟工具

```ts
simulateTransaction(txData)
estimateGas(txData)
previewAssetChange(txData)
```

作用：

- 模拟交易是否会成功
- 估算 gas
- 预览交易前后的资产变化
- 避免盲目执行危险交易

#### D. 权限控制工具

```ts
checkSpendingLimit(agentId, token, amount)
checkAllowedRecipient(toAddress)
checkAllowedContract(contractAddress)
checkDailyLimit(agentId, token, amount)
recordAuditLog(action)
```

作用：

- 限制 Agent 单笔可花金额
- 限制 Agent 每日总额度
- 限制 Agent 可交互地址
- 限制 Agent 可交互合约
- 记录所有执行行为，方便审计

## AgentScoope Wallet 项目启发

AgentScoope Wallet 可以设计成一个受限执行钱包：

> Agent 可以帮助用户查询资产、分析风险、模拟交易，甚至在权限范围内执行链上操作；但 Agent 的执行能力必须受到严格限制。

安全边界：

- 单笔交易金额限制
- 每日总额度限制
- 收款地址白名单限制
- 合约交互白名单限制
- 高风险交易必须用户二次确认
- 所有操作必须记录审计日志

项目表达可以总结为：

> AgentScoope Wallet 不是让 AI 直接控制资产，而是让 AI 在用户预先定义的额度、对象、时间和方法边界内执行小额链上操作。Web3 负责提供可验证账户、链上交易、权限模块和审计记录，AI 负责理解用户目标、生成操作计划、调用工具并解释执行结果。
