真实钱包连接仍然保留在控制台和高级路径里，但不再作为落地页体验的前置门槛。

## 今日学习主题：Hackathon Demo 要先降低进入门槛

今天最大的收获是：

> Web3 Agent 产品的 Demo 入口，不一定要从连接钱包开始；如果目标是讲清楚产品价值，应该先让用户进入可看的 Demo，再逐步解释真实钱包和执行边界。

PactTrader 的核心不是“连接钱包”本身，而是：

- 用户先定义 Pact / policy 边界；
- Agent 只能生成受限策略或交易建议；
- CAW 执行层只在授权范围内执行；
- 越界动作会被拒绝；
- 所有 proposal、decision、execution result 都能被审计。

所以落地页更适合作为产品价值入口：先用 Try Demo 让用户看到完整路径，再在控制台里解释 CAW、EOA、Agent Wallet 和 Pact 的关系。

## Hackathon 推进

今天的推进对 Hackathon Demo 很关键，因为它把产品第一印象从“请先连接钱包”改成了“先试 Demo”。

这样做的好处：

1. **评审路径更短**  
   评委打开落地页后，可以直接进入 Dashboard，不需要先安装 / 连接钱包。

2. **更容易讲清楚 PactTrader 的核心价值**  
   进入 Demo 后再解释 Agent Wallet、Pact、CAW、风控和审计，比在首页直接要求连接钱包更顺。

3. **避免钱包环境成为阻塞点**  
   Hackathon 展示时，网络、钱包插件、测试网资产都可能造成干扰。预置 Demo 模式可以先保证核心叙事可跑。

4. **真实执行路径仍然可扩展**  
   Try Demo 不等于放弃真实钱包；它只是把真实 EOA / CAW 配对放到更后面的高级路径里。
