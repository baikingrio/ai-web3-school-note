# AI × Web3 Concept Helper｜可交互学习小工具

> WCB 任务：Week 1｜AI 向任务｜完成 AI 可交互学习产物  
> 目标：做一个最小可交互学习产物，帮助学习 AI / Web3 概念。

## 它解决什么学习问题

我在 AI × Web3 School 学习时，经常会遇到一个问题：

> 单独看概念都懂一点，但很难把它和自己的项目、风险边界、下一步行动联系起来。

所以我做了一个很小的交互工具：用户输入一个 AI / Web3 概念、自己的背景和困惑点，工具会生成：

1. 一句话解释；
2. 一个生活类比；
3. 和 AgentScoope Wallet / AI × Web3 项目的关系；
4. 一个安全提醒；
5. 一个自测问题；
6. 一个下一步行动。

它不是完整产品，更像一个“学习概念解释器 + 任务检查卡片生成器”。

## 交互方式

### 方式 A：网页 demo

直接打开：

```text
experiments/ai-web3-concept-helper/index.html
```

输入：

- 想学习的概念；
- 当前背景；
- 最困惑的问题。

点击生成后，会在页面里生成学习卡片。

### 方式 B：CLI 小工具

```bash
cd experiments/ai-web3-concept-helper
python3 concept_helper.py --concept "智能账户" --background "我在做 AgentScoope Wallet" --question "它和 EOA、多签有什么区别？"
```

## 输入示例

```text
概念：智能账户
背景：我在做 AgentScoope Wallet，一个受限 Agent 钱包项目
困惑：智能账户和普通 EOA、多签到底差在哪里？
```

## 输出示例

```text
学习卡片：智能账户

一句话解释：智能账户可以理解成“带规则的钱包账户”，它不只靠一个私钥直接控制，而是可以通过合约逻辑设置权限、限额、恢复和自动化策略。

生活类比：EOA 像一把单人钥匙；多签像公司账户，需要几个人一起确认；智能账户像一把带规则的智能门锁，可以设置谁能开、什么时候能开、最多能做什么。

和项目的关系：在 AgentScoope Wallet 里，智能账户 / Safe 可以承载资产和权限；Agent 不是 owner，只能作为受限角色，在白名单、预算和方法范围内执行低风险动作。

安全提醒：不要把主私钥交给 Agent。适合自动化的是低风险、测试网、白名单、预算内操作；提高额度、改 owner、主网资金和无限 approve 必须人工确认。

自测问题：如果 Agent 想把 5 USDC 额度提高到 50 USDC，这应该是自动执行、人工确认，还是直接拒绝？为什么？

下一步行动：把这个概念映射到自己的项目，写清楚“谁能发起、谁能批准、谁承担风险、如何撤销”。
```

## 哪部分由 AI 生成，哪部分由我人工修改 / 验证

- **AI 辅助生成**：初始解释结构、生活类比、学习卡片模板、测试输入输出示例。
- **我人工修改 / 验证**：把内容改成和自己的 AgentScoope Wallet 项目相关；检查 EOA、智能账户、多签、Safe、policy、human confirmation 等表述是否准确；删除可能造成误解的“自动执行主网资金”等危险表达。
- **代码实现**：用一个不依赖 API key 的本地 HTML / Python 小工具实现交互，避免提交任何 token、私钥或 `.env`。

## 限制

1. 目前不是接入真实 LLM 的产品，而是一个最小可交互 demo，输出来自预设的 AI 辅助模板和规则。
2. 它适合学习和梳理概念，不适合直接作为投资、签名或链上操作建议。
3. 对复杂概念的解释还比较粗，需要后续加入更多概念库和案例。
4. 没有读取链上实时数据，也不会发起交易。
